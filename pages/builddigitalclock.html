How to Build a Digital Clock

When I was first planning this project, I was going to make a digital timer. As I got further into building it, I realized that before you learn how to walk, you should really learn how to crawl.

Dates and times in JavaScript are not as straight forward as you would believe them to be, so I decided to start at the beginning and create a digital clock. Of course, I decided to have some fun with it and spice it up by employing a few fun CSS tricks.

If you build this project, don't delete it because I plan on using it as the base for more projects in the future.

What's nice about this project is that there is hardly any HTML. It basically consists of 2 DIVs and a button. The first DIV will be the container for the project, the second DIV will be where we display the time, and the button will be what calls out the digital clock (the second DIV).

[id/class = smart-phone-container] || Create a div that will serve as the container of the project. We are going to style this to look like a smartphone when we get to the CSS.

[id/class = build-digital-clock] || Create a div that will house the digital clock. Put in a placeholder time (11:11:11) just so you have an idea of what you are working with.

[id/class = smart-phone-home-button] || Lastly, create the button. In many cases people would use another div for this but with accessibility becoming such a big focus, it's better to become accustomed to using the correct elements. Note: I still have a lot to learn about writing semantic HTML so it is definitely something that may change as I build more projects.

That's it! Close the container DIV and then we can move onto the CSS!

This CSS for this project is definitely more extensive than the JavaScript but with a website/app it's how everything works together to make something nice. Never underestimate the importance of good CSS because it will always be what the user notices before anything else.

This is is the main container for the project that makes it look like a smart phone. It will hold the other divs and components you will be using for the project.

The very first thing you should set is the position of the div. The position of relative is like telling the browser that other things are going to be put in here, and it will be used as the footprint of where those things can be situated.

Set the overflow to hidden so anything that finds it's way outside of the div will not show up in the browser

The next 3 things were for the margin in my project. They basically just give you some space and center the container in the div above this current one.

The background settings are all pretty standard. The background is linked to a URL that will use a photo as a background instead of a color. If you would like to download the exact picture I used, you can grab it here on Unsplash. You can also use any picture you would like but I suggest using one that has a portrait orientation

The other background settings are used for positioning the background in the div. The most significant one is having a background size of "contain." Generally you would like to use "cover" but we would like the full picture showing to get the effect of the smartphone in this project.

I have set the height and width here. I tend to use REM because I have been told that it is better for accessibility over pixels. I do believe it is also more responsive.

The rest of the settings are for detailing. If you're not away yet, border-radius is what gives you those nice little rounded edges you see on every button on every single website.

The ".smart-phone-home-button" is the round button that exists... well used to exist on every single smartphone. In this project, pushing it will pull out the time bar. Generally it would bring up the home screen but hey, I needed something to work with.

The first thing you need to do is set the position to "absolute". Think of a position of absolute as unlocking the element to move around wherever you want on the page. By default it will set itself to the top left corner but with a few settings we can put it wherever we would like.

What I recommend doing first is setting the width and height of the button. This will allow us to calculate the absolute center of the div (if you would like to place the button in the center). You can pretty much put it anywhere you want by playing with the percentages in the next steps.

The left setting is how far we will place the element from the left side of the parent div (.digital-clock-container). The "calc" measurement is a little hack that allows you to find center since 50% will give center the left-most side of the button and not the center of the it. What this does is takes the 50% measurement and then you want to subtract half of the button element's width. Since you set it at 28px in the last step, you would cut that in half and minus 14px here.

The bottom measurement is how far off of the bottom of the parent div your button will sit.

The rest of the settings are used for styling. You may have noticed that I removed the outline (accessibility folks are fuming at me) but this is something we will look at in an upcoming step. I promise, this will be accessible!

Cursor pointer makes the little pointy hand appear when you hover over the element. Do not underestimate how much this is needed!

[.smart-phone-home-button:active] || This next little CSS trick I cannot take credit for. I 100% stole it from Wes Bos (who also has one of the best courses for learning JS called JavaScript30 available for free on his site!). What this trick does is scale the button that you click down just slightly to give it the appearance of being physically clicked.

[.smart-phone-home-button:focus] || Alright! I told you I was going to make the button accessible. When you have an outline on a round button, it ends up being square and there is no way to adjust the border-radius. Using box-shadow will accomplish the same thing as outline but you can make it round. Add some styling and you got yourself a slick looking (and accessible) button!

Now we would like to build the div that is going to house the digital clock. Right now the div should have a placeholder time in it, but when we finish the JS it will be a clock that adjusts based on the current time of the day. Use the placeholder text to help you style it.

The first thing, like the button, is to set the position of the div to "absolute" so we can move it wherever we would like.

Now you're probably looking at the next setting (right) and wondering "Why does that say -80%? Well, because we would like our time to slide in from off of the screen, this is actually going to position the div outside of the div on the right-hand side. For styling purposes, leave it set to 0 for now but change it to -80% when you're done.

The bottom of 20% will place this div 20% up from the bottom of the parent div.

The next 3 settings are merely used for the positioning of the text. Set the display to flex to make the div flex activated. set justify-content to center to center the text horizontally, and align-items to center to center everything vertically.

The background color setting is set to black (rgb(0,0,0)) but the final setting (0.6) is what controls the opacity of the div. This is what makes the div slightly transparent.

Everything else is pretty self explanatory and widely used, with the exception of transition. The transition setting is what is going to make your div sliding in from the right look cool and not just appear out of nowhere. This transition will take effect when the "right" setting is changed. It will take 0.5s to run through it's animation and the animation will be ease-in-out.

Don't forget to set the "right" setting back to -80% if you haven't done so already!

[.build-digital-clock.clicked] || The .clicked added onto the end of the build.digital.clock class (.build.digital.clocked.clicked) will be a class that is added with the JavaScript. Everything that happens in this section is when this div also contains the class of "clicked."

This is where that "right" setting from the last section makes some magic happen. If you haven't noticed yet, the "right" setting of the clicked class will change what the "right" setting is in the last step. So instead of being -80% and off the screen, the right side of the div will now sit at 0% and fully visible inside the div. This will all happen when the "clicked" class is added in the JavaScript.

The transition effect here is what will happen when you call the time in. It will take effect when "right" is changed, it will take 1 second to animate, and the animation will be ease-in-out.

[.build-digital-clock p] || This will simply remove the padding and margin from the paragraph tag that we generate in the JS.

That is ALL of the CSS!
